Key Features

Parallel Processing

All user profiles are processed concurrently using asyncio.gather(), meaning the pipeline never waits for one user before starting the next.

Resilient by Design

Each user is processed in an independent try/except block. If one profile fails, the pipeline logs the error and continues with the rest.

Retry Logic

LLM calls use exponential backoff retries. Transient API errors are handled automatically before giving up on a user.

Provider Agnostic

Switching LLM providers requires changing only two lines in config.py. No changes to pipeline logic or prompt structure needed.
